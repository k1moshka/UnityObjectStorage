Описание каждого коммита должно содержать тэг и описание изменямых аспектов без глагола.
Например, 
 New: интерфейс генератора сущностей

 - тэг New подразумевает добавление нового функционала, поэтому глагол "добавление" не нужен
  т.е. New: добавление интерфейса генератора сущностей - не правильное описание коммита

Список тэгов:
 - New - Добавление нового функционала
 - Fix - Фикс текущейго функционала, в том числе баг фикс, изменение поведения, рефакторинг который влечет изменение функционала либо АПИ
 - Clean - Очистка утаревшего кода, ненужных комментариев, форматирование кода, рефакторинг без изменения функционала
 - Doc - Документирование кода (XML документация, комменты, осисательные текстовики, UML)
